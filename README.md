# Serial Port Tapper + Protocol Analyzer

**Bahasa Indonesia** | **English below**

---

## Deskripsi / Description

**ID:** Tool untuk menyadap (tap) komunikasi serial dan mempelajari protokol secara otomatis tanpa dataset training awal. Dirancang khusus untuk fuel pump dispenser (Sanki, Wayne, Gilbarco, Tokheim, dll) namun bisa dipakai untuk protokol RS-232/RS-422/RS-485 apapun.

**EN:** A serial port tapping tool with automatic protocol learning — no prior training data needed. Designed for fuel pump dispensers (Sanki, Wayne, Gilbarco, Tokheim, etc.) but works with any RS-232/RS-422/RS-485 protocol.

## Quick Start

### 1. Install dependency

```bash
pip3 install pyserial

# Optional: untuk clustering yang lebih akurat
pip3 install scikit-learn numpy
```

### 2. Konfigurasi port

Edit `run_tap.sh`, bagian `KONFIGURASI`:

```bash
PORTS=(
    "/dev/ttyUSB0:SankiPump:9600:RS232"
)
LOG_FORMAT="hex"   # WAJIB hex untuk analyze.py
```

### 3. Jalankan

```bash
chmod +x run_tap.sh

./run_tap.sh start    # mulai tapping di background
./run_tap.sh status   # cek status
./run_tap.sh stop     # berhenti + simpan log
./run_tap.sh analyze  # analisis protokol dari log
```

## tap.py — Serial Port Tapper

### Fitur

- Tap beberapa port sekaligus dengan konfigurasi individual
- Deteksi arah TX/RX: `auto`, `rs485`, `pattern`, `size`, `alternating`, `none`
- Packet assembly dengan timeout configurable per port
- Log ke file dengan variabel `{date}`, `{datetime}`, dll
- Hardware-aware detection untuk RS-485/Modbus RTU

### Format konfigurasi port

```
/dev/port:Label:Baudrate:DataBit:Parity:StopBit:Type:Detection:Timeout(ms)
```

**Contoh:**

```bash
# RS-232 sederhana
python3 tap.py -p /dev/ttyUSB0:Pump1:9600

# RS-485 Modbus RTU
python3 tap.py -p /dev/ttyUSB0:Modbus:19200:8:E:1:RS485:rs485:30

# Multi-port
python3 tap.py \
  -p /dev/ttyACM0:POS:115200:8:N:1:RS232:pattern:50 \
  -p /dev/ttyUSB0:Pump:9600:8:N:1:RS232:auto:100
```

### Detection modes

| Mode | Cara kerja | Cocok untuk |
|------|-----------|-------------|
| `auto` | Kombinasi semua method (recommended) | RS-232, RS-422 |
| `rs485` | Hardware-aware, analisis Modbus frame | RS-485 |
| `pattern` | Deteksi dari pola ASCII (AT+, OK, GET) | AT commands |
| `size` | Adaptive size-based (command pendek, response panjang) | General |
| `alternating` | Toggle TX→RX→TX sederhana | Debug |
| `none` | Semua dicatat sebagai RX | Passive monitor |

## analyze.py — Protocol Pattern Analyzer

Mempelajari protokol secara otomatis dari log hasil tapping. **Tidak butuh dataset training awal** — data dari tapping itu sendiri yang jadi sumber pembelajaran.

### 5 Layer Analisis

```
L1. Timing Analysis
    Gap waktu antar packet → tentukan TX/RX
    Gap panjang (>threshold) = TX (master initiator)
    Gap pendek (<threshold) = RX (slave response)

L2. Frame Detection
    Scan byte yang konsisten di semua packet:
    - Preamble sync bytes (FC FC FC, AA AA, dll)
    - Start byte: STX(0x02), ENQ(0x05), SOH(0x01)
    - End byte: ETX(0x03), EOT(0x04)
    - Length byte position

L3. Command Clustering
    N-gram prefix grouping (1-3 byte pertama)
    → temukan berapa jenis command yang berbeda
    → guess fungsi tiap command dari ukuran & arah

L4. Data Field Detection (BCD)
    Scan valid BCD byte di tiap posisi:
    - BCD byte: semua nibble 0-9 (bukan A-F)
    - Identifikasi: price_per_unit, volume, amount, counter
    - Static field (sama antar packet) = header/address
    - Variable field (berubah antar packet) = data transaksi

L5. Checksum Validation
    Test XOR, LRC, Two's Complement LRC
    → verifikasi frame structure yang ditemukan L2
```

### Usage

```bash
# Analisis log file
python3 analyze.py tapping_20260302_120000.txt

# Simpan ke JSON
python3 analyze.py tapping.txt -o protocol_map.json

# Analisis hex string manual (investigasi satu packet)
python3 analyze.py --hex "FC FC FC 02 01 00 00 00 48 00 00 27 10 72 99 E2 E3"

# Via run_tap.sh
./run_tap.sh analyze
./run_tap.sh analyze-hex "FC FC FC 02 01 ..."
```

### Akurasi vs jumlah packet

| Jumlah packet | Yang bisa dianalisis |
|--------------|----------------------|
| 1–2          | Frame structure (preamble, start byte) |
| 3–9          | Command grouping, partial BCD |
| 10–19        | Checksum, timing threshold |
| 20–49        | Static vs variable fields |
| 50+          | Full protocol map, field interpretations akurat |

## Use Case: Fuel Pump Dispenser (Sanki)

### Karakteristik protokol Sanki (dari hasil analisis)

```
Preamble  : FC FC FC (3 sync bytes)
Start byte: 0x02 (STX)
Encoding  : BCD untuk data numerik
Struktur  : [FC FC FC][STX][ADDR][...data...][CHKSUM][END?]

Field yang teridentifikasi:
  offset 11–12 : Price per unit  (BCD, Rp/Liter)
  offset 22–23 : Volume / Amount (BCD)
```

### Strategi capture untuk hasil optimal

```
Sesi 1 (30 menit):
  → Capture polling rutin + beberapa transaksi
  → analyze.py → temukan frame structure + command types

Sesi 2 (1 jam, lebih banyak transaksi):
  → Validasi temuan sesi 1
  → Bedakan field static (header) vs variable (data)
  → Identifikasi field volume, amount, price

Sesi 3+:
  → Confidence score naik
  → Decode transaksi real-time makin akurat
```

### Multi-merek

Karena analyze.py belajar dari data tanpa asumsi protokol, tool ini kompatibel dengan berbagai merek:

| Merek | Interface | Status |
|-------|-----------|--------|
| Sanki | RS-232 | Terdeteksi preamble FC FC FC |
| Wayne / Dresser | RS-232/RS-485 | Supported |
| Gilbarco Veeder-Root | RS-232 | Supported |
| Tokheim | RS-232 | Supported |
| Generic Modbus RTU | RS-485 | Full support (hardware-aware) |

## Troubleshooting

**Permission denied saat buka port:**
```bash
sudo usermod -a -G dialout $USER
# logout → login ulang
```

**Log file tidak terbaca analyze.py:**
```bash
# Pastikan format hex di run_tap.sh:
LOG_FORMAT="hex"
```

**BCD field tidak terdeteksi:**
```bash
# Pastikan cukup packet (minimal 20+)
# Capture beberapa transaksi lengkap
```

**analyze.py lambat:**
```bash
# Uninstall scikit-learn untuk pakai pure Python mode
pip3 uninstall scikit-learn numpy
```
